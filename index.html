직원들과 대시보드 공유하기
아래 5가지 방법 중 상황에 맞는 방법을 선택하세요. 방법 1 (가장 쉬움)부터 시작하는 것을 추천합니다.
1
📁 파일 직접 공유 (가장 쉬움)
⭐ 난이도: 쉬움 · 추천
방법: index.html 파일 하나를 이메일·카카오톡·네이버웍스로 전송
장점: 별도 설정 없음, 수신자가 파일을 받아 Chrome으로 열면 바로 동작
단점: 매주 업데이트된 파일을 다시 전송해야 함

  1. C:\비버웍스\대시보드시스템\index.html 파일을 카카오톡/이메일로 전송
  2. 수신자가 파일을 저장 후 Chrome으로 열기
  3. 직원이 직접 ERP Excel 파일을 업로드하면 본인 화면에서 분석 가능
2
🖥️ 사내 네트워크 서버 (실시간 공유)
⭐⭐ 난이도: 보통 · 같은 WiFi 환경 필요
방법: 영업본부장 PC에서 서버를 실행하면 같은 사내 WiFi의 모든 직원이 접속 가능
장점: 링크 하나로 전체 공유, 실시간 업데이트, 추가 비용 없음
단점: 서버 PC가 켜져 있어야 함, 같은 네트워크에서만 접속 가능

🚀 실행 방법:
# 방법A: 서버시작.bat 더블클릭 (이 폴더에 포함)
# 방법B: 명령프롬프트에서 직접 실행
cd C:\비버웍스\대시보드시스템
python -m http.server 8080
서버 실행 후 → 직원들에게 아래 주소 카톡으로 전송:
# 본인 PC IP 확인: 명령프롬프트에서 ipconfig 입력
http://[본인PC_IP]:8080
# 예: http://192.168.1.100:8080
3
☁️ Netlify Drop (무료 웹 호스팅, 5분)
⭐⭐ 난이도: 보통 · 외부 접속 가능
방법: Netlify 무료 서비스를 이용해 인터넷 어디서나 접속 가능한 URL 발급
장점: 무료, 빠름, 외부 인터넷에서도 접속 가능 (재택근무자 포함)
단점: 민감 데이터는 직원이 직접 업로드해야 함 (서버에 데이터 저장 안됨)

  1. Chrome에서 netlify.com/drop 접속
  2. C:\비버웍스\대시보드시스템 폴더 전체를 페이지로 드래그
  3. 자동으로 URL 발급 (예: https://amazing-beaver.netlify.app)
  4. 발급된 URL을 직원들에게 카카오톡으로 공유
  5. 매주 업데이트 시 같은 방법으로 재배포 (1분 소요)
4
📂 구글 드라이브 / 네이버 클라우드 공유
⭐ 난이도: 쉬움
방법: 드라이브에 파일 업로드 후 링크 공유 (직원이 다운로드 후 실행)

  1. 구글 드라이브 또는 네이버 클라우드 접속
  2. index.html 파일 업로드
  3. 공유 링크 생성 (링크가 있는 모든 사용자에게 공개)
  4. 링크를 단체 카톡방/이메일로 공유
  5. 직원이 링크에서 파일 다운로드 → Chrome으로 열기
💡 팁: 매주 같은 파일에 덮어쓰기 업로드하면 링크가 유지됩니다.
5
🐙 GitHub Pages (영구 URL, 개발자 추천)
⭐⭐⭐ 난이도: 어려움 · 가장 안정적
방법: GitHub 무료 계정으로 고정 URL 발급 — 영구적으로 사용 가능
장점: 무료, 고정 URL, 버전 관리, 매우 안정적

  1. github.com 무료 가입
  2. 새 Repository 생성 (예: beaver-dashboard)
  3. index.html 파일 업로드
  4. Settings → Pages → Branch: main 선택 → Save
  5. URL 자동 발급: https://[계정명].github.io/beaver-dashboard

방법난이도비용외부접속실시간공유추천 상황
1. 파일 직접 공유⭐ 쉬움무료✅❌빠른 공유, 소규모 팀
2. 사내 네트워크 서버⭐⭐ 보통무료❌ 사내만✅사내 WiFi 환경, 실시간
3. Netlify Drop ⭐추천⭐⭐ 보통무료✅주간배포재택+사무실 혼합 팀
4. 구글/네이버 드라이브⭐ 쉬움무료✅❌이미 드라이브 사용 중
